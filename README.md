## üìñ Overview

Flexifun_ATG is a full-stack web application built as a Game-Based Learning Platform specifically tailored for autistic children. The platform aims to provide an engaging, safe, and effective environment for learning and development through interactive games and educational content. It features a robust backend for user management, game logic, and data storage, coupled with a dynamic and responsive frontend for an optimal user experience.

This project leverages modern web technologies to deliver real-time interactivity, rich animations, and a secure infrastructure, addressing the unique needs of its target audience.

## ‚ú® Features

-   **Interactive Game-Based Learning**: Engaging educational games designed for cognitive development.
-   **User Authentication & Authorization**: Secure signup, login, and role-based access control (JWT).
-   **Real-time Interactions**: Live communication features powered by Socket.io for collaborative or dynamic gameplay.
-   **Progress Tracking & Analytics**: Visualize learning progress and performance using interactive charts (Chart.js).
-   **Secure Payment Integration**: Seamless handling of subscriptions or in-app purchases via Stripe.
-   **Cloud-based Asset Management**: Efficient storage and delivery of game assets and user media using Cloudinary.
-   **Animated & Responsive UI**: Modern, intuitive, and accessible user interface with smooth animations (GSAP, Framer Motion) and responsive design (Tailwind CSS).
-   **Comprehensive API**: Robust RESTful API endpoints for managing users, games, progress, and administrative tasks.
-   **Email Notifications**: Automated email sending for important events and updates (Nodemailer).
-   **Data Validation & Security**: Extensive input validation and security middleware to protect against common web vulnerabilities.

## üöÄ Quick Start

Follow these steps to get Flexifun_ATG up and running on your local machine.

### Prerequisites

Ensure you have the following installed:

-   **Node.js**: `v18.x` or higher (recommended: `v20.x`)
-   **pnpm**: Package manager (`npm install -g pnpm`)
-   **MongoDB**: An instance of MongoDB (local or cloud-hosted like MongoDB Atlas)

### Installation

1.  **Clone the repository**
    ```bash
    git clone https://github.com/Anup158/Flexifun_ATG.git
    cd Flexifun_ATG
    ```

2.  **Install dependencies**
    This project uses `pnpm` as its package manager.
    ```bash
    pnpm install
    ```

3.  **Environment setup**
    Create a `.env` file in the root directory by copying the example:
    ```bash
    cp .env.example .env
    ```
    Open `.env` and configure your environment variables. The key variables you'll need are:

    | Variable                      | Description                                                  | Default      | Required |
    | :---------------------------- | :----------------------------------------------------------- | :----------- | :------- |
    | `PORT`                        | Port for the backend server                                  | `5000`       | Yes      |
    | `MONGO_URI`                   | MongoDB connection string                                    |              | Yes      |
    | `JWT_SECRET`                  | Secret key for signing JWTs                                  |              | Yes      |
    | `CLOUDINARY_CLOUD_NAME`       | Your Cloudinary cloud name                                   |              | Yes      |
    | `CLOUDINARY_API_KEY`          | Your Cloudinary API key                                      |              | Yes      |
    | `CLOUDINARY_API_SECRET`       | Your Cloudinary API secret                                   |              | Yes      |
    | `STRIPE_SECRET_KEY`           | Your Stripe secret key                                       |              | Yes      |
    | `STRIPE_WEBHOOK_SECRET`       | Your Stripe webhook secret for event handling                |              | No       |
    | `EMAIL_USER`                  | Email address for Nodemailer (e.g., Gmail user)              |              | No       |
    | `EMAIL_PASS`                  | Password/App-specific password for the email user            |              | No       |
    | `CLIENT_URL`                  | URL of the frontend application (e.g., `http://localhost:5173`) |              | Yes      |
    | `SALT_ROUNDS`                 | Number of salt rounds for bcrypt hashing                     | `10`         | No       |

4.  **Database setup**
    Ensure your MongoDB instance is running. The application will connect to the `MONGO_URI` specified in your `.env` file. No explicit migration commands are needed; Mongoose will handle schema creation on first connection based on your models.

5.  **Start development servers**
    Open two separate terminal windows.

    **Terminal 1 (Backend Server):**
    ```bash
    pnpm run dev:server
    ```
    The backend server will start on `http://localhost:[PORT]` (default `http://localhost:5000`).

    **Terminal 2 (Frontend Application):**
    ```bash
    pnpm run dev:client
    ```
    The frontend development server will start on `http://localhost:5173` (default Vite port).

6.  **Open your browser**
    Visit `http://localhost:5173` to access the application.

## üöÄ Deployment

### Production Build

To create a production-ready build for both client and server:

```bash
pnpm run build:client
pnpm run build:server
```
This will generate optimized bundles in the `dist` directories for both frontend and backend.

### Deployment Options

-   **Netlify**: The `netlify.toml` file is configured for seamless deployment of the frontend to Netlify. After building the client (`pnpm run build:client`), you can deploy the `dist` folder.
-   **Docker**: The presence of `.dockerignore` indicates potential Dockerization. You can create a `Dockerfile` for both the client (if served statically by a web server like Nginx) and the server (Node.js application) for containerized deployment.
-   **Server Hosting**: The built server application can be deployed to any Node.js compatible hosting service (e.g., Vercel, Railway, Render, AWS EC2, DigitalOcean).

## üìö API Reference

The backend API is built with Express.js and provides RESTful endpoints for various functionalities.

### Authentication

Users can authenticate using JWT (JSON Web Tokens). Upon successful login, a JWT is issued, which must be included in subsequent requests for protected routes (e.g., in the `Authorization` header as `Bearer <token>`).

### Endpoints

A high-level overview of expected endpoint categories:

-   `POST /api/auth/register`: Register a new user.
-   `POST /api/auth/login`: Authenticate user and get JWT.
-   `GET /api/users/me`: Get current user's profile.
-   `PUT /api/users/me`: Update current user's profile.
-   `GET /api/games`: Retrieve a list of available games.
-   `GET /api/games/:id`: Get details of a specific game.
-   `POST /api/progress`: Submit game progress/results.
-   `GET /api/progress/:userId`: Get progress data for a specific user.
-   `POST /api/payments/checkout`: Initiate a Stripe checkout session.
-   `POST /api/payments/webhook`: Stripe webhook listener for payment events.
-   `GET /api/socket`: Establishes a WebSocket connection for real-time features.
